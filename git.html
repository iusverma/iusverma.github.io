Git commands & shortcuts

I have listed down all the git commands which I have worked on so far and also some of the trick commands which I have used in during my day to day job

  • Git command general

    Here are all the git commands I have used so far
    1. Cloning using SSH
    git clone git@github.com:iusverma/iusverma.github.io.git
    2. Cloning using HTTPS
    git clone https://github.com/iusverma/AutomatedParkingSystem.git
    Note: When you clone a repository, it creates a default reference "origin" in ur local. This origin will always point to the repo you first cloned.
    3. Checking the local repository
    git remote -vv
    Output Sample:
    origin git@github.com:iusverma/AutomatedParkingSystem.git (fetch)
    origin git@github.com:iusverma/AutomatedParkingSystem.git (push)
    4. Setting multiple repository inside same structure
    git remote add new_repo_name remote_repo_url
    E.g. git remote add upstream git@github.com:iusverma/iusverma.github.io.git
    Note: Adding upstream will create an additional link with name "upstream", which will link to this new repo.
    Output sample for git remote -vv after above command
    origin git@github.com:iusverma/AutomatedParkingSystem.git (fetch)
    origin git@github.com:iusverma/AutomatedParkingSystem.git (push)
    upstream git@github.com:iusverma/AutomatedParkingSystem.git (fetch)
    upstream git@github.com:iusverma/AutomatedParkingSystem.git (push)
    5. Checking the local branch
    git branch
    6. Checking all the branches local & remote
    git branch -a
    7. Checking which branch current branch is tracking to
    git branch -vv
    8. Create new branch locally
    git checkout -b branch_name
    E.g. git checkout -b release-1.0
    9. Pushing and tracking locally created branch
    git push origin branch_name
    git branch --set-upstream-to origin/branch_name
    10. Create new branch from remote
    git checkout -b branch_name origin/branch_name
    E.g. git checkout -b develop origin/develop
    11. Alternate/switch branch
    git checkout branch name
    12. Pulling and pushing code from remote
    12.1.
    git pull
    git push
    - this will pull/push work from/to default tracking branch
    12.2. (recommended by me)
    git fetch origin - fetches all changes accross repository
    git pull --rebase - mergse all changes from remote and put local changes on top
    git push origin branch_name - push changes to specifc branch
    13. Comming a code
    git add file name
    git commit -m "commit message"
    14. Reverting a commit
    git reset --soft HEAD^
    15. Changing commit message for a commit already pushed
    git rebase -i HEAD~N
    - N is the one commit before the commit which you want to change.
    - Once the editor opens, edit the file and put e/r infront of the commit you want to change.
    - save file and exit
    git push -f origin branch name
    16. Checking current state
    git status -uno
    - show only the tracked files
    - do not show new files
    git status
    - shows all files
    17. Merging multiple commits into one
    git rebase -i HEAD~N
    - N number of last commits you want to merge + 1
    - Once the editor is open, edit the file, keep first one as P/Pick, mark all other as "S"
    - save file and exit
    18. Stashing or stacking local changes
    git stash save "stash message"
    19. Check the stash bucket
    git stash list
    20. Applying stashed changes
    git stash apply stasg@{N}
    - N being stash number you want to apply
    21. Picking some one elses commit
    git cherry-pick commit-id
    22. Picking some one elses commit without commiting it
    git cherry-pick -n commit-id
    23. Creating a local repo and pushing to github
    1. git init
    2. git add remote
    3. git push origin
