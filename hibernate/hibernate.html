    Join strategies

    Hibernate provides 3 basic strategies for storing table with inheritance. Let's see this with example. For exampl, we have following classes.
    public class Vehicle{
      private int id;
      private String name;
    }
     And following are two classes inherited from Vechicle class

    1. Overriding the default method

    public class TwoWheeler extends Vehicle {
      private int id;
      private String steeringHandle;
    }
     And
    public class FourWheeler extends Vehicle{
      private String steeringWheel;
    }
     Now here are the three styles of table based on the table strategy chosen.

  • Single Table

    In this style, there will always a single table for all hierarchy and all properties from super to child class will form a column in table. There will be a Descriminator Type (dtype) column which will contain the class name to identiy records seperately.
    For the classes above here is the table structure

  • Table Per Class

    In this style, there will one single table for each concrete class in inheritance hierarchy and each table will have correspodning class properties and properties from super class.
    For the classes above here is the table structure

  • Joined

    In this style, there will one single table for each concrete class in inheritance hierarchy. However properties for the super class will only be present in corresponding table. For child class tables, only those column will come which are defined as properties in child class. Also there will be a id column from the super class table. For the classes above here is the table structure.
