  • Default Method

    Starting from JAVA 8, interface can have a implementation for a method in it. This means, if class implementing this interface doesn't override the method, default method from interface will be called.
    Here is an example of default method in an interface.
    public interface Loggable{
      public void log(){
        System.out.println("Loggable.log");
      }
    }
     Now this interface can be inherited in following two manners.

    1. Overriding the default method

    public class Logger1 implements Loggable{
      public void log(){
        System.out.println("Logger1.log");
      }
    }

    2. Without overriding default method

    public class Logger2 implements Loggable{
      public void anotherLocalMethod(){
        // TODO Something;
      }
    }
     However this may lead to diamond problem. In it an interface is extended by two different interfaces and then a third class implements this new interface. In this case which implementation third class will use. For example here is another interface which extends Loggable interface.
    public class Logger3 implements Loggable{
      public void log(){
        System.out.println("Logger3.log");
      }
    }
     Now, if we have another class extending and implementing Logger1 and Logger3, following will be the situation.
    public class Logger extends Logger1 implements Logger3{
      public void print(){
        System.out.println(log()); // This will be a problem, JVM doesn't know which log (Logger1.log or ) to call.
      }
      public static void main(String []args){
        Logger logger = new Logger();
        logger.print();
      }
      // Following code need to be uncommented for class to compile
      /*
      public void log(){
        System.out.println("Logger3.log");
      }
      */
    }
     Fortunately JAVA compiler will not let the code compile unless local implementation for log is present. Hence uncomment the last section in above code for compilable version.
